üéØ Problem Statement

Understanding and interpreting audio data is pivotal in numerous applications, from urban planning to personalized user experiences. This project embarks on a dual mission:

  • Acoustic Environment Classification: Determine whether a 7-second audio clip was recorded indoors or outdoors.
  • Gender Classification: Identify the speaker's gender (male or female) based on audio recordings.

Both tasks leverage the MLEnd London Sounds dataset, enriched with manually labeled gender data, to train supervised machine learning models.

üîç Why This Matters

  • Complexity of Audio Data: Audio signals are time-series data with intricate patterns. Deciphering these patterns requires sophisticated feature extraction and analysis.
  • Real-world Applications:
    • Acoustic Environment: Enhances context-aware systems, aids in urban noise monitoring, and improves hearing aid functionalities.
    • Gender Classification: Supports personalized interfaces, sociolinguistic studies, and demographic analytics.

üõ†Ô∏è Machine Learning Pipeline

1. Data Acquisition

  • 2,498 audio clips (7 seconds each) from various London locations
  • Metadata includes: area, spot, environment, participant ID
  • Manual labeling of gender due to missing demographic data

2. Feature Extraction

Utilized librosa library to extract the following spectral features:

  • MFCC (Mel-Frequency Cepstral Coefficients): Captures timbral properties
  • Spectral Bandwidth: Measures frequency spread
  • Zero-Crossing Rate: Counts sign changes in waveform
  • Tempo: Estimates audio beat speed

3. Data Preprocessing

  • Skewness Reduction: Log, square, and cube root transformations
  • Standardization: Features scaled to mean 0 and variance 1

4. Model Training & Tuning

  • Split: 70% training / 30% testing
  • Models Used: Random Forest, SVM, KNN, Logistic Regression
  • Hyperparameter Tuning: Performed using RandomizedSearchCV

5. Evaluation Metrics

  • Accuracy
  • Precision
  • Recall
  • F1-Score
  • Confusion Matrix
