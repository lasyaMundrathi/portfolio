COVID-19 Impact on London Underground Ridership

Objective

The objective of this project is to predict the number of taps (N) at London Underground stations using machine learning, focusing on XGBoost Regressor for time-series forecasting. The goal is to aid public transport optimization through accurate prediction models.

Background

The dataset from Transport for London (TfL) spans from Jan 1, 2020 to Apr 30, 2021, covering daily entries/exits at 15-minute intervals. It includes station metadata, lockdown phases, and public service closures. Understanding how events impact travel behavior helps improve service delivery and urban planning.

Dataset Highlights

  • Temporal resolution: 15-minute intervals
  • Features include station name, borough, lockdown phases, closures (schools, pubs, shops)
  • Target variable: N (number of taps)

Machine Learning Pipeline

  • Data Preprocessing: Fill missing values, label encode categorical features, and convert dates.
  • Feature Engineering: Lag features for 7, 14, and 21-day intervals; new date-based fields (month, quarter, is_weekend); one-hot and mean encoding.
  • Model Selection: Main model: XGBoost; benchmarks: Random Forest, SVM, Linear Regression.
  • Validation: Used TimeSeriesSplit (6-fold) to preserve time dependencies. Metrics: RMSE, MAE, RÂ².
  • Hyperparameter Tuning: Applied RandomizedSearchCV for optimal model settings.

Feature Descriptions

  • CALENDARDATE: Date of entry/exit
  • HOUR: Quarter-hour represented in decimals (e.g., 9.25)
  • ENTRYEXIT: Whether the movement was an entry or exit
  • STATIONTYPE: Categories like Shopping, Termini, Suburb
  • LOCKDOWN_NATIONAL / REGIONAL: Indicators of restriction levels

Modeling Approach

  • Lagged features added by station and hour to retain temporal grouping context
  • Split into train/validation/test sets based on date to avoid leakage
  • XGBoost trained using reg:squarederror objective
  • Rolling-window cross-validation across 6 folds
